# RoboScene+ Reconstruction Diagnosis & Fix Summary

Generated by debug_reconstruction.py

## Confirmed Root Causes (ranked by impact)

### 🔴 #1 CRITICAL — Batch boundary 180° pose flips
- **Evidence:** Rotation ratio 3.95x at boundaries, literal 180° flips at frames 249→250, 119→120, 459→460
- **Cause:** VGGT processes each batch of 10 independently. With 511 still photos (not video), consecutive shots face different directions. Batch k+1 has zero constraint relative to batch k.
- **Effect:** 51 disconnected local maps oriented arbitrarily → explosion
- **Fix:** `--overlap 5` in run_vggt.py (overlapping batches + Procrustes stitching) ✅ IMPLEMENTED

### 🔴 #2 CRITICAL — Double-scaling: cameras.bin fx=16,453 (7.78x too large)
- **Evidence:** `cameras.bin` fx=16,453 = fx_json(2,114) × resize_ratio(7.784) exactly
- **Cause:** `scale_intrinsics()` scaled VGGT's 518px focal to 4,032px. Then `write_colmap_reconstruction()` scaled it AGAIN by max(4032,3024)/518=7.784. Result: 7.78x over-scaled focal → HFOV=14° instead of 87°.
- **Effect:** All 3D points initialise at 7.78x wrong depth. Training diverges.
- **Fix:** `colmap_utils.py` now uses already-scaled K directly (no second multiply) ✅ IMPLEMENTED
- **Fix:** `scale_intrinsics()` now uses uniform scale: `fx=fy=fx_vggt*max(W,H)/518` ✅ IMPLEMENTED

### 🔴 #3 CRITICAL — Non-square pixels: fx/fy=1.333 baked into intrinsics
- **Evidence:** `scale_intrinsics()` used scale_x=W/518=7.784, scale_y=H/518=5.838 → different fx and fy
- **Cause:** 4032×3024 (4:3) resized to 518×518 (1:1). VGGT predicts single symmetric focal. Non-uniform back-scaling produces fx≠fy, violating square-pixel constraint.
- **Effect:** Every pixel's depth→3D unprojection uses wrong aspect geometry.
- **Fix:** `scale_intrinsics()` now enforces `fx=fy=fx_vggt*max(W,H)/518` ✅ SAME FIX AS #2

### ✅ NOT A BUG — Pose convention (c2w vs w2c)
- **Evidence:** Round-trip test: `inv(images.bin w2c) == camera_poses.json c2w` to 2e-7 precision
- VGGT `extrinsic_4x4` IS w2c. Written directly to `images.bin` — correct.
- `gsplat_colmap_dataset.py` reads w2c, inverts to c2w for training — correct.

### ⚠️ WARNING — All 100,000 points have empty tracks
- **Effect:** None — gsplat init uses XYZ positions (present), not tracks.
- **Fix:** Optional — assign `track=[(image_id, 0)]` per point.

### ⚠️ WARNING — PINHOLE model for ultrawide lens
- VGGT estimated HFOV=87° vs true iPhone 0.5x ~120°
- Barrel distortion causes ~30° FOV underestimate
- **Fix (future):** Pre-undistort images with Apple's k1/k2 before VGGT
